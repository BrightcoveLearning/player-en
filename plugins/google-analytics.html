Overview

The Brightcove Google Analytics plugin provides functionality to send data to Google Analytics.

Implement using Players module

To implement the Google Analytics plugin using the Players module, follow these steps:

  1. In Studio select the Players module.
  2. Click the player to which you wish to add the Google Analytics plugin.
  3. In the Player's property screen select:
    1. Third-Party Analytics -> Google Analytics
    2. Check the Enable Google Analytics
    3. Click Sign in to Google Account
  4. Complete the sign in process by entering information for your Google account and which Google Analytics account to send data.
  5. Finish the configuration using the provided form.
  6. Click Save.

Implement using code

To implement a plugin the player needs to know the location of the plugin code, a stylesheet if needed, the plugin name and plugin configuration options. The location of the plugin code and stylesheet are as follows:

Options

The following plugin options are used when configuring the plugin:

  • tracker: Google Analytics tracker ID
  • eventsToTrack: An object that lists the events to send and the name that will be used for those events.
    • Example: {player_load: 'My name for player load event'}
  • paramsToTrack: An object that lists the params (aka custom dimensions) to include in events and the index to track them as. Use the format dimension1, dimension2, etc. based on the "index" field in GA. Do not use the dimension name.
    • Example: {bcvideo_account: 'dimension1'}
  • debug: true to use GA debug url and visualize console trace. This should not be used in production environment.

Events and custom dimensions

Any event name used by the player's analytics_event and analytics-beacon events can be used, but these ones have special handling in the GA plugin:

  • player_load
  • video_engagement
  • video_impression
  • play_request
  • ad_start
  • ad_end
  • error

These events are not shown in the Studio UI, but could be enabled with manual configuration.

  • video_view
  • video_complete

The following are custom dimensions available for use:

  • bcvideo_account
  • bcvideo_platform_version
  • bcvideo_player
  • bcvideo_range
  • bcvideo_session
  • bcvideo_tags
  • bcvideo_video_duration
  • bcvideo_video_seconds_viewed
